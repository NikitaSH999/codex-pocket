# 🚀 codex-pocket 0.0.2v

> _Когда лежишь на кроватке, но хочешь чтобы AI работал за тебя_

Мобильное продолжение локального `Codex` с телефона.  
Открыл PWA, ткнул по сессии, одобрил действие, отправил сообщение, снова откинулся на подушку.

Это всё ещё не "чатик в браузере".  
Это **пультик для твоего локального Codex**, только теперь собранный заметно взрослее, чем `0.0.1`.

---

## 🧠 Зачем это вообще

Есть вещи, которые сначала выглядят как "ну прикольная штука", а потом ты без них уже не хочешь работать.

`codex-pocket` как раз из таких:

- 🛏️ Лежишь на кровати и не хочешь возвращаться к ноуту
- 👀 Видишь, что Codex делает прямо сейчас
- 💬 Продолжаешь тред с телефона
- ✅ Approvals, команды, файлы, input requests, форки, resume
- 🗂️ Быстро прыгаешь между воркспейсами и сессиями

Как говорится: _"почему это не встроено по умолчанию?"_

## 🏗️ Как это теперь устроено

`0.0.1` был дерзкой первой атакой.

`0.0.2v` уже разложен нормально:

```text
┌─────────────────────────────┐
│  📱 Телефон / PWA           │
└──────────────┬──────────────┘
               │ HTTP + WebSocket
┌──────────────▼──────────────┐
│  codex-bridge               │
│  Express + JSON-RPC bridge  │
└──────────────┬──────────────┘
               │ WebSocket JSON-RPC
┌──────────────▼──────────────┐
│  codex app-server           │
│  настоящий локальный Codex  │
└─────────────────────────────┘
```

Монорепа состоит из:

- `apps/codex-bridge` - локальный bridge на машине с Codex
- `apps/mobile-pwa` - мобильный интерфейс
- `packages/shared` - общие типы и контракты
- `docs` - архитектура, сетап, безопасность и заметки

То есть идея та же самая, но кости уже крепче.

## ⚡ Quick start

Самый быстрый путь:

```bash
npm install
npm run build
npm run dev:bridge
npm run dev:pwa
```

Если хочешь сценарий поближе к "как оно будет жить":

```bash
npm run build
npm run start:bridge
```

После этого открываешь адрес bridge с телефона, вводишь PIN и идёшь дальше по жизни.

---

## 🎯 Первый запуск

1. Запускаешь bridge на ПК
2. Открываешь PWA с телефона
3. Смотришь PIN в консоли bridge
4. Pair trusted phone
5. Выбираешь сессию или воркспейс
6. Ложишься обратно на кроватку и работаешь

Да, основной use case по-прежнему именно такой.

---

## 🧪 Что именно изменилось после 0.0.1

По сути это уже новая версия идеи, а не просто маленький апдейт:

- bridge и mobile-клиент разделены явно
- логика завязана на текущий `codex app-server` flow
- лучше оформлены approvals и live events
- появились shared types вместо разрозненных структур
- документация стала честнее и взрослее
- акцент теперь на LAN / Tailscale, а не на "давайте просто откроем это наружу и посмотрим"

`0.0.1` был "прикольно, это уже работает".  
`0.0.2v` больше про "окей, теперь из этого реально можно делать нормальный продукт".

## 🔌 Где это должно жить

Нормальный режим:

- локальная сеть
- Tailscale
- один доверенный телефон

Не нормальный режим:

- открытый интернет без дополнительной защиты
- публичная админка на весь мир
- ожидание, что это заменит remote desktop

Если коротко: штука для личного Codex, а не для поднятия мини-SaaS в один клик.

## 🤓 Технически

Под капотом сейчас:

- локальный child-process lifecycle для `codex app-server`
- WebSocket JSON-RPC клиент
- HTTP API + `/events` stream для телефона
- pairing через PIN и trusted-device token
- timeline и session mapping из app-server threads
- PWA-сборка для мобильного сценария

Да, оно всё ещё лёгкое. Но уже не выглядит как случайный прототип за ночь.
