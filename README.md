# Codex Switchboard

Локальный mobile-first webui для Codex, который открывается с телефона, живет у тебя дома или по белому IP и работает не как очередной чатик к API, а как тонкий клиент к настоящему `codex app-server`.

Короче:

- пишешь в `localapp` -> агент работает в `localapp`
- пишешь в `VPN` -> агент уходит в `VPN`
- смотришь с телефона -> видишь чат, activity, команды, tool calls и состояние сессии

Если в чате уже пишут "ого, интересненько", значит вайб пойман правильно.

## Что это вообще

Это не обертка ради обертки.

Это локальный switchboard для Codex:

- с PIN-логином
- с workspace-aware историей
- с живой activity-лентой
- с `Plan mode`
- с быстрыми `/`-actions
- с запуском через `.bat`

Идея простая: не сидеть все время только в desktop app, а иметь удобную web-морду, которую можно открыть хоть с Arc, хоть с телефона, хоть с другого устройства в локалке.

## Что уже умеет

- Fastify backend с cookie-сессией и PIN
- React webui с `Chat / Activity / Sessions / Settings`
- bridge к локальному `codex app-server`
- стрим сообщений и activity по WebSocket
- импорт старой истории из `~/.codex/sessions`
- выбор папки через встроенный folder browser
- привязка сессий и истории к конкретному workspace
- quick menu и slash actions для быстрых действий
- private-режим и отдельный remote-режим для белого IP

## Что пока не one-to-one с desktop Codex

Parity пока не 100%.

Пока нет:

- полноценного file upload UI
- полного desktop slash-комплекта
- отдельного `Speed` / reasoning picker
- нормального approval UI
- полного менеджера MCP shortcuts как в desktop

Но для локальной работы и сценария "взять Codex с собой на телефон" уже очень даже жирно.

## Быстрый старт

```bash
npm install
npm run dev
```

Dev-режим поднимет:

- frontend: `http://localhost:5173`
- backend: `http://localhost:4318`

Production:

```bash
npm run build
npm start
```

Или по-человечески:

```bat
start-codex-mobile-webui.bat
```

Для белого IP:

```bat
start-codex-mobile-webui-remote.bat
```

## Первый вход

1. Открываешь страницу.
2. На первом запуске задаешь PIN.
3. Логинишься этим PIN с телефона, ноута или другого браузера.
4. Заходишь в нужный workspace и продолжаешь работу там.

Теперь логика именно такая: workspace в UI = контекст, в котором стартует новая сессия.

## Как этим пользоваться без боли

### 1. Workspace board

Слева есть board по проектам.

Если видишь `localapp`, `VPN` или любую другую папку:

- можешь открыть историю именно этого проекта
- можешь создать новую сессию именно в нем
- можешь импортировать старый тред и продолжить работу уже в webui

### 2. Chat + Activity

В `Chat` идет обычный диалог.

В `Activity` видно, что агент реально делает:

- commentary
- команды
- tool calls
- статусы
- плановые блоки

То есть это не магическая черная коробка, а нормальный live feed.

### 3. Slash menu

В поле ввода набираешь `/` и получаешь quick actions:

- `Plan mode`
- `MCP status`
- `Workspace board`
- `Settings`
- переключение на конкретный workspace
- создание новой сессии в выбранной папке

Плюс есть удобства для ежедневного использования:

- `Ctrl+Enter` отправляет сообщение
- `Enter` по slash-команде запускает первое подходящее действие
- `Esc` закрывает quick/slash меню

## Локально и по белому IP

По умолчанию приложение живет в private-режиме:

- loopback
- LAN
- локальные private subnet

Если нужен доступ по статическому внешнему IP:

- запускай `start-codex-mobile-webui-remote.bat`
- он включает public-mode
- добавляет inbound rule в Windows Firewall
- пробует сделать UPnP port mapping

Если роутер не дает UPnP или провайдер держит тебя за CGNAT, магии не будет. Тут уже либо ручной проброс, либо другая сеть.

## Почему это не очередной кринжовый pet project

Потому что штука реально решает боль:

- не надо сидеть только в одном desktop-клиенте
- можно видеть, что делает агент
- можно быстро перепрыгивать между проектами
- можно продолжать работу со старых Codex threads
- можно открыть все с телефона и проверить прогресс без RDP и без плясок
